Model Context Protocol (MCP)

9.1 What MCP Is and Why It Matters

The Model Context Protocol (MCP) is an open standard, introduced by Anthropic in late 2024, for connecting AI models to external tools and data sources. Before MCP, every AI integration was bespoke: a different API shape, a different auth mechanism, a different data format for every tool. MCP defines a universal connector interface.

The analogy: MCP is to AI tools what USB is to peripherals. Any USB device works with any USB port. Any MCP server works with any MCP-compatible AI client. This standardisation is the most significant architectural development in AI tooling since the transformer.

9.2 MCP Architecture

MCP has three components: a host (the AI application, e.g., Claude Desktop, your custom app), a client (the MCP client layer inside the host), and one or more servers (external processes that expose tools, resources, and prompts).

  • Tools: functions the model can call (search, execute code, query database)
  • Resources: data sources the model can read (files, database tables, API endpoints)
  • Prompts: pre-defined prompt templates the model can invoke
