Advanced Agent Patterns

14.1 The ReAct Pattern

ReAct (Reason + Act) is the foundational pattern for tool-using AI agents. The model alternates between reasoning (producing a thought) and acting (calling a tool or producing a final answer). The key insight: explicit reasoning traces improve tool use accuracy and make debugging possible.

14.2 Circuit Breakers for AI Dependencies

When an AI API is down or degraded, your automation should stop calling it rather than piling up failing requests. The circuit breaker pattern tracks failure rates and opens (stops calls) when the failure rate exceeds a threshold.

14.3 Graceful Degradation

When the primary AI model is unavailable, what should your automation do? The three options: fail hard (return error), degrade gracefully (use a faster/cheaper/local fallback), or queue for later. Most production systems want the second or third option.

14.4 Context Window Management at Scale

Context window limits are a hard constraint in every AI automation system that maintains conversation history. The strategies for managing context at scale:

Sliding Window

Keep only the last N messages. Simple to implement. Loses older context entirely. Appropriate when recent context is all that matters.

Summary Compression

Periodically summarise older messages into a single context message. Preserves meaning at the cost of an additional API call. Appropriate for long-running sessions.

Hierarchical Memory

Important messages (marked by user or detected by importance classifier) are preserved verbatim. Unimportant messages are dropped or summarised. The most sophisticated approach — and the one the Akasha system implements.
