LangChain, LlamaIndex, and AI Orchestration Frameworks

8.1 Why AI Orchestration Frameworks Exist

LangChain and LlamaIndex emerged to solve the same problem: the gap between 'call an AI API' and 'build a production AI application'. Connecting to vector stores, managing conversation history, chaining multiple AI calls, routing between tools — these patterns repeat across every AI application. The frameworks provide them pre-built.

The important caveat: these frameworks add abstraction. Abstraction is valuable when it hides complexity you don't need to control. It is costly when it hides behaviour you do need to understand. Know what the framework is doing before trusting it in production.

8.2 LangChain: Composable AI Chains

LangChain's core abstraction is the chain: a sequence of components (prompt templates, models, output parsers, tools) that can be composed with the pipe operator. The LCEL (LangChain Expression Language) syntax makes this readable.

8.3 LlamaIndex: Retrieval-Augmented Generation

LlamaIndex is specialised for retrieval-augmented generation (RAG): indexing documents, generating embeddings, storing them in a vector database, and retrieving relevant chunks at query time. This is the standard architecture for 'ask questions about your documents' use cases.

8.4 LangGraph: Stateful Agent Architectures

LangGraph is the most important framework for complex AI automation. Where LangChain chains are linear (A → B → C), LangGraph supports cycles, conditionals, and persistent state — the structures needed for real agents that loop, branch, and remember.

The mental model: LangGraph is a state machine where each node is a function that reads the state, does work (possibly calling an LLM), and returns updates to the state. The graph defines transitions — which node runs next based on current state.

8.5 CrewAI and AutoGen: Multi-Agent Frameworks

CrewAI and AutoGen provide higher-level abstractions for multi-agent systems — collections of AI agents with different roles and capabilities working together on a task. These are appropriate when a task genuinely requires specialisation (one agent researches, another writes, another edits) and when the coordination overhead is justified by the task complexity.
