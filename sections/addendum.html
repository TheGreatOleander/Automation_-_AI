Addendum: The Whole Word on Building Systems

This manual has grown from its first edition because the field has grown. The tools are better, the patterns are more mature, and the failure modes are better understood. But the discipline has not changed.

The Code Volleyball pattern reflects something important about how AI-assisted engineering actually works in practice: not as a single oracle that produces correct systems on demand, but as a relay of perspectives, each one catching what the others miss. The human orchestrates. The models build. The ledger remembers.

Observability, structured outputs, local model deployment, vector store operations, agent evaluation — each of these sections exists because a practitioner hit its absence as a wall. The wall is now a door.

The one thing that does not change: automation built without understanding its failure modes will eventually fail in production, in ways its builders did not anticipate, at the worst possible time. The checklists, the runbooks, the eval suites — these are not bureaucracy. They are the difference between automation that helps and automation that surprises.

Build deliberately. Instrument everything. Know when to hand back to a human.

— The Compiler
