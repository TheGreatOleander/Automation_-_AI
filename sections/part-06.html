Bash and Python: The Craft Layer

6.1 Environment Management

AI automation projects span multiple environments: local development, CI, staging, production. Each has different secrets, different endpoints, different resource limits. Managing this cleanly from the start prevents the 'works on my machine' failures that destroy production trust.

direnv: Per-Directory Environment

python-dotenv Pattern

6.2 Async Python for AI Automation

Synchronous AI calls are the bottleneck in most automation pipelines. A single thread calling an API sequentially wastes 95% of its time waiting. Async Python allows dozens of concurrent API calls with no threading complexity.

6.3 Mocking AI for Testing

Running your full test suite against the live AI API costs money, is slow, and is non-deterministic. Mock the AI client during tests. Your tests should verify the automation logic — routing, validation, persistence, error handling — not the model's output.

6.4 Debugging Non-Deterministic Systems

AI outputs are probabilistic. A failing automation might fail 1 in 20 runs. Debugging this requires different techniques than deterministic software.

Deterministic Replay

Log every AI call with its full input and output. When a failure occurs, you can replay the exact call that caused it. This turns a probabilistic failure into a deterministic one.

Seed Control for Sampling Parameters

Most AI APIs support a seed parameter for deterministic output (same seed + same prompt = same output, modulo model updates). Use seeds in testing and when debugging specific failures.
